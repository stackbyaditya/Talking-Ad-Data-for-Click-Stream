# Click Stream Analysis for Bot Detection Using Machine Learning and Deep Learning

## 1. Project Overview
This repository implements an end-to-end behavioural bot detection workflow derived from the TalkingData AdTracking Fraud Detection dataset. The project is motivated by click fraud in large-scale digital advertising systems, where invalid traffic degrades attribution quality, inflates campaign costs, and biases downstream bidding and ranking algorithms. In mobile advertising ecosystems, fraudulent click activity can be generated by malware, emulators, headless browsers, scripted traffic brokers, or stealthy automation frameworks that mimic legitimate user interactions.

Traditional fraud detection pipelines often rely on static signatures such as IP blacklists, velocity caps, or manually engineered heuristics. These approaches are brittle because adversaries continuously modify request timing, network routing, and device-level fingerprints. This project therefore adopts a behavioural analytics perspective: instead of only asking whether a click was attributed, it models how a session behaves over time. The implemented workflow transforms event-level ad-click records into session-level behavioural descriptors, augments them into a balanced multi-class bot detection dataset, constructs synthetic temporal sequences for sequence learning, and evaluates both tabular machine learning models and temporal deep learning architectures.

The repository is structured as a reproducible research pipeline rather than only a collection of experiments. It contains: (i) a Kaggle-to-session transformation stage, (ii) synthetic behavioural data generation with controlled class balance, (iii) advanced behavioural feature engineering, (iv) tabular preprocessing and boosting-based classification, (v) temporal sequence generation for recurrent and convolutional models, and (vi) visualization and reporting artifacts. These components provide enough technical grounding to support drafting an IEEE-style manuscript on behavioural click fraud detection.

## 2. Problem Statement
Let an observed clickstream be represented by an event sequence
\[
\mathcal{E}=\{e_i\}_{i=1}^{N},
\]
where each event \(e_i\) contains a network-device signature, application identifiers, and a timestamp. The objective is to infer whether the behaviour associated with a click session corresponds to a legitimate human user or to one of several bot categories. In this repository, the final supervised formulation is a three-class classification problem:

\[
f: \mathbf{x} \rightarrow y,\quad y \in \{0,1,2\},
\]

where \(y=0\) denotes `human`, \(y=1\) denotes `moderate_bot`, and \(y=2\) denotes `advanced_bot`.

The core challenge is that individual click events are weak signals. Bots can replicate surface-level metadata such as browser type, geolocation, and even coarse click frequencies. Therefore, the problem is reformulated from event classification to behavioural session classification. Each training instance encodes interaction statistics such as interval entropy, burstiness, movement variability, and idle patterns. A second representation transforms each session into a fixed-length sequence tensor for temporal modelling. The project thus addresses two related tasks:

1. Session-level tabular classification using aggregated behavioural descriptors.
2. Sequence-level classification using synthetic temporal interaction trajectories derived from session statistics.

## 4. Feature Engineering
The repository engineers behavioural, temporal, network/device, and heuristic features.

### 4.1 Behavioural Features
These features are intended to approximate human motor variability and cursor irregularity in the absence of raw mouse trajectories:

| Feature | Interpretation |
|---|---|
| `mouse_speed_mean` | Average cursor speed proxy |
| `mouse_speed_std` | Dispersion of mouse speed |
| `mouse_path_length` | Approximate path length traversed during the session |
| `direction_change_count` | Number of trajectory directional changes |
| `movement_std` | General movement variability |
| `coordinate_entropy` | Randomness of spatial interaction behaviour |

Additional advanced behavioural descriptors are computed in [`preprocessing/behavioral_feature_engineering.py`](/c:/Users/Aditya%20Kumar/Desktop/talkingdata-adtracking-fraud-detection/preprocessing/behavioral_feature_engineering.py):

\[
\texttt{movement\_acceleration}=\frac{\texttt{mouse\_speed\_std}}{\texttt{mouse\_speed\_mean}+10^{-5}},
\]
\[
\texttt{mouse\_direction\_entropy}=\frac{\texttt{direction\_change\_count}}{\texttt{mouse\_path\_length}+1},
\]
\[
\texttt{trajectory\_smoothness}=\frac{\texttt{mouse\_path\_length}}{\texttt{direction\_change\_count}+1}.
\]

These measures approximate whether movement is smooth and linear (more bot-like) or noisy and irregular (more human-like).

### 4.2 Temporal Features
Temporal dynamics encode request cadence and session burst structure:

| Feature | Interpretation |
|---|---|
| `session_duration_sec` | Total elapsed duration of the session |
| `request_interval_mean` | Mean inter-request delay |
| `request_interval_std` | Variability of inter-request delay |
| `clicks_per_minute` | Click intensity normalized by time |
| `requests_per_minute` | Request rate proxy |
| `success_rate` | Approximate conversion rate |
| `burstiness` | Ratio of interval dispersion to interval mean |
| `click_interval_entropy` | Entropy of discretized temporal intervals |

The advanced stage adds:

\[
\texttt{click\_burst\_score}=\frac{\texttt{clicks\_per\_minute}}{\texttt{request\_interval\_mean}+1},
\]
\[
\texttt{session\_idle\_ratio}=\frac{\texttt{request\_interval\_std}}{\texttt{session\_duration\_sec}+1},
\]
\[
\texttt{interaction\_variability}=\frac{\texttt{mouse\_speed\_std}+\texttt{request\_interval\_std}+\texttt{click\_interval\_entropy}}{3},
\]
\[
\texttt{behavioral\_complexity}=\texttt{movement\_std}+\texttt{coordinate\_entropy}+\texttt{interaction\_variability}.
\]

### 4.3 Network and Device Features
The synthetic pipeline preserves or simulates contextual metadata:

| Feature | Interpretation |
|---|---|
| `browser` | Browser family |
| `operating_system` | Device operating system |
| `device_type` | Mobile, desktop, or tablet |
| `country` | Country-level geolocation |
| `region` | Region/province placeholder |
| `is_proxy` | Proxy-use indicator |

### 4.4 Heuristic and Anomaly Features
Two higher-level fraud indicators are also used:

1. `bot_likelihood_score`, computed in the realistic generator as a weighted combination of normalized request rate, burstiness, anomaly score, and \(1-\texttt{success_rate}\).
2. `anomaly_score`, derived via `IsolationForest` over selected behavioural-temporal features.

These features provide weak supervisory priors while preserving the multi-class behavioural framing.

## 5. Data Preprocessing Pipeline
The tabular preprocessing workflow is implemented in [`preprocessing/preprocess_dataset.py`](/c:/Users/Aditya%20Kumar/Desktop/talkingdata-adtracking-fraud-detection/preprocessing/preprocess_dataset.py). The main steps are:

1. Load [`final_training_dataset_advanced.csv`](/c:/Users/Aditya%20Kumar/Desktop/talkingdata-adtracking-fraud-detection/data/processed/final_training_dataset_advanced.csv).
2. Validate the absence of missing values.
3. Remove duplicate rows if present.
4. Drop metadata columns from the modelling matrix:
   `session_id`, `ip_address`, `user_agent`, `label_name`, `source_click_time`, and `source_attributed_time`.
5. Split features into numeric and categorical subsets.
6. Apply `RobustScaler` to numeric features and `OneHotEncoder(handle_unknown="ignore")` to categorical features.

`RobustScaler` is appropriate because behavioural clickstream features exhibit strong skew, long tails, and outlier-heavy distributions. Median- and interquartile-range-based scaling is less sensitive to anomalous sessions than standard z-score normalization, which is important for click fraud analysis where outliers are often semantically meaningful.

The earlier Kaggle transformation stage also saves a preprocessing artifact at [`artifacts/preproc_pipeline.pkl`](/c:/Users/Aditya%20Kumar/Desktop/talkingdata-adtracking-fraud-detection/artifacts/preproc_pipeline.pkl). That artifact records 13 numeric session-aggregation features and 5 categorical Kaggle columns (`ip`, `app`, `device`, `os`, `channel`) used at the raw-session conversion stage.

## 6. Sequence Generation
Temporal sequence generation is implemented in [`preprocessing/session_sequence_generator.py`](/c:/Users/Aditya%20Kumar/Desktop/talkingdata-adtracking-fraud-detection/preprocessing/session_sequence_generator.py). Since the final modelling dataset is session-level rather than raw event-level, the repository synthesizes a fixed-length temporal representation from aggregate statistics.

Each session produces a sequence matrix of length 25 and dimensionality 15. The final tensor stored in [`model_outputs/lstm_training_data_v3.npz`](/c:/Users/Aditya%20Kumar/Desktop/talkingdata-adtracking-fraud-detection/model_outputs/lstm_training_data_v3.npz) has shape:

\[
9000 \times 25 \times 15.
\]

The sequence features are:

| Sequence feature |
|---|
| `mouse_speed_mean` |
| `mouse_speed_std` |
| `movement_std` |
| `coordinate_entropy` |
| `clicks_per_minute` |
| `requests_per_minute` |
| `request_interval_mean` |
| `request_interval_std` |
| `click_burst_score` |
| `behavioral_complexity` |
| `interaction_variability` |
| `session_idle_ratio` |
| `mouse_speed_delta` |
| `click_event` |
| `pause_event` |

At each timestep, values such as mouse speed and request interval are sampled from Gaussian distributions centered on session statistics. Binary `click_event` is drawn from a Bernoulli distribution with probability \(\min(\texttt{clicks\_per\_minute}/60, 1)\). `pause_event` is set when the sampled interval exceeds the session-specific 75th percentile. The sequence generator validates that no NaNs are present and rejects constant sequences.

Sequences are useful because they encode local temporal context unavailable in aggregated tabular representations. Even though these sequences are synthetic rather than directly observed logs, they enable evaluation of convolutional and recurrent models under a consistent tensor format.

## 7. Machine Learning Models
The tabular classification stage is implemented in [`Model1/models/train_boosting_models.py`](/c:/Users/Aditya%20Kumar/Desktop/talkingdata-adtracking-fraud-detection/Model1/models/train_boosting_models.py).

### 7.1 Random Forest
Random Forest is an ensemble bagging method that aggregates many decorrelated decision trees. For a given sample \(\mathbf{x}\), the final prediction is obtained by majority vote across trees. In this repository, the model uses 300 trees and maximum depth 15. It is robust to nonlinear interactions and mixed feature types, which makes it a strong baseline for behavioural tabular data.

### 7.2 XGBoost
XGBoost implements additive gradient boosting:
\[
\hat{y}^{(t)} = \hat{y}^{(t-1)} + f_t(\mathbf{x}),
\]
where each new tree \(f_t\) is fitted to reduce the residual error of the current ensemble. The repository uses 400 estimators, learning rate 0.05, maximum depth 6, subsampling 0.8, and column sampling 0.8.

### 7.3 LightGBM
LightGBM also performs gradient boosting but grows trees leaf-wise rather than level-wise. This often improves accuracy and efficiency on structured, high-dimensional tabular features. The implemented configuration uses 400 estimators, learning rate 0.05, and 31 leaves.

Tree-based methods perform well here because the dataset is structured, relatively compact, and dominated by engineered session descriptors rather than raw unstructured signals. Decision-tree ensembles can exploit thresholded interactions among burstiness, interval entropy, anomaly score, and device/network context without requiring very large sample sizes.

## 8. Deep Learning Models
The sequence modelling stage is implemented in [`Model1/models/deep_learning/train_dl_models.py`](/c:/Users/Aditya%20Kumar/Desktop/talkingdata-adtracking-fraud-detection/Model1/models/deep_learning/train_dl_models.py), [`Model1/models/deep_learning/dl_model_architectures.py`](/c:/Users/Aditya%20Kumar/Desktop/talkingdata-adtracking-fraud-detection/Model1/models/deep_learning/dl_model_architectures.py), and [`Model1/models/deep_learning/dl_utils.py`](/c:/Users/Aditya%20Kumar/Desktop/talkingdata-adtracking-fraud-detection/Model1/models/deep_learning/dl_utils.py).

### 8.1 CNN
The CNN model uses two `Conv1D` blocks with 64 and 128 filters followed by max pooling, flattening, a dense hidden layer, and softmax output. It is designed to detect local temporal motifs such as click bursts, movement spikes, and pause signatures.

### 8.2 LSTM
The LSTM model stacks a 64-unit recurrent layer with `return_sequences=True` and a 32-unit recurrent summarization layer. This architecture models sequential dependencies such as slow-fast-pause transitions and evolving request cadence.

### 8.3 CNN-LSTM
The CNN-LSTM hybrid first applies convolution and pooling to extract short-range behavioural motifs, then feeds the compressed sequence into an LSTM layer. This corresponds to local-pattern extraction followed by temporal integration.

### 8.4 CNN-BiLSTM
The CNN-BiLSTM architecture uses convolutional front-end layers followed by a bidirectional LSTM. The bidirectional layer processes the sequence in both temporal directions, which can capture symmetric patterns such as pause-click-pause or acceleration-deceleration motifs.

### 8.5 CNN-Attention-LSTM
This model uses a convolutional front-end, an LSTM with sequence output, a self-attention layer, global average pooling, and a dense classifier. The attention stage weights informative timesteps more strongly, which is relevant when only a subset of the 25 timesteps contains highly suspicious behaviour.

## 10. Evaluation Metrics
For true positives \(TP\), false positives \(FP\), true negatives \(TN\), and false negatives \(FN\), the repository uses:

\[
\text{Accuracy}=\frac{TP+TN}{TP+TN+FP+FN},
\]
\[
\text{Precision}=\frac{TP}{TP+FP},
\]
\[
\text{Recall}=\frac{TP}{TP+FN},
\]
\[
F_1 = 2\cdot\frac{\text{Precision}\cdot\text{Recall}}{\text{Precision}+\text{Recall}}.
\]

For multi-class classification, the implementation reports weighted precision, weighted recall, weighted \(F_1\), and one-vs-rest weighted ROC-AUC. Confusion matrices provide class-wise error structure, showing where human sessions are confused with moderate or advanced bot behaviour.

## 12. Key Observations
1. Boosting models outperform the implemented deep learning baselines on the current dataset. XGBoost achieves the best overall accuracy and ROC-AUC.
2. Deep learning models still achieve strong ROC-AUC values around 0.964–0.966, indicating meaningful class ranking ability even when hard classification performance lags the tree ensembles.
3. The synthetic sequence tensor provides a valid sequence-learning benchmark, but it does not add enough temporal signal to outperform tabular aggregation. This is consistent with the fact that the sequences are generated from session summaries rather than raw event logs.
4. Behavioural feature distributions indicate that human sessions exhibit larger `session_idle_ratio`, higher positive `interaction_variability`, and higher `behavioral_complexity`, while bot classes show higher `click_burst_score` and `mouse_direction_entropy`.
5. The original Kaggle sample contains mostly single-click sessions. Therefore, the strongest project contribution lies in behavioural dataset construction rather than in extracting naturally occurring long-horizon temporal dynamics from the raw benchmark alone.

## 13. Limitations
The main limitation is synthetic data generation. Session trajectories, cursor dynamics, and temporal sequences are generated from designed stochastic processes rather than observed human-computer interaction logs. This improves controllability but weakens ecological validity. In addition:

- the public Kaggle sample has very limited multi-click continuity
- the sequence tensor is generated from session summaries, not native event trajectories
- no raw cursor coordinates or DOM interaction logs are available
- network/device metadata are partly simulated rather than collected from production traffic

These constraints should be acknowledged explicitly in any paper derived from the repository.

## 14. Future Work
Future improvements include:

1. Replace synthetic sequence generation with real event-level interaction traces.
2. Incorporate raw cursor trajectories, dwell time, and scroll dynamics.
3. Introduce adversarial bot simulators that mimic human burst-pause structures more faithfully.
4. Extend the pipeline to online or streaming bot detection for near-real-time fraud prevention.
5. Explore domain adaptation and self-supervised pretraining for behavioural embeddings.
